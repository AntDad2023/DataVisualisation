01

初始化与 option —— ECharts 的核心思维

ECharts 与 D3 最大的区别：你不操作 DOM，只写一份"配置对象"（option），ECharts 负责渲染一切。

// 第一步：准备一个有宽高的 div
// <div id="chart" style="width:600px;height:300px"></div>

// 第二步：初始化
const chart = echarts.init(document.getElementById('chart'))

// 第三步：写 option，setOption 一次搞定所有
chart.setOption({
  title:   { text: '我的第一个图表' },
  tooltip: { trigger: 'axis' },   // 悬停提示
  xAxis:   { data: ['Mon','Tue','Wed','Thu','Fri'] },
  yAxis:   {},                        // 自动推断范围
  series:  [{ type: 'bar', data: [120,200,150,80,170] }]
})
💡核心思维：ECharts = "声明你想要什么"。option 是一份 JSON 描述，告诉 ECharts"我要什么图、数据是什么、轴怎么配"，渲染细节全部交给 ECharts 内部处理。
02

图表类型 —— series.type

ECharts 只需改一个字段 series.type 就能切换图表类型。数据格式不变，类型随意切。

// 改 type 就换图，数据格式相同时甚至可以混用
series: [
  { type: 'bar',     data: [...] },  // 柱状图
  { type: 'line',    data: [...] },  // 折线图
  { type: 'scatter', data: [...] },  // 散点图
  { type: 'pie',
    data: [{ name:'A', value:30 }, ...] },  // 饼图（格式稍不同）
]
// 同一个图表里可以放多个 series，实现"柱+线"混合图
💡一个 option 里可以有多个 series，每个 series 可以是不同类型，ECharts 自动叠加渲染。这是 ECharts 比 D3 方便得多的地方。
03

坐标轴配置 —— xAxis / yAxis

ECharts 的轴是声明式的：写清楚类型和数据，刻度、间距、标签自动计算。常见类型有 category（分类轴）和 value（数值轴）。

xAxis: {
  type: 'category',                    // 分类轴：等间距排列
  data: ['一月','二月','三月','四月'],
  axisLabel: { rotate: 30 }            // 标签旋转，防重叠
},
yAxis: {
  type: 'value',                       // 数值轴：自动计算范围
  name: '销售额（万元）',
  splitLine: { lineStyle: { type:'dashed' } }
}
// 还有 time（时间轴）和 log（对数轴）
💡轴的 type 非常关键：category 用于文字标签（产品名/月份），value 用于连续数值，time 用于时间序列，log 用于数量级差距很大的数据。
04

数据集 —— dataset + encode

dataset 是 ECharts 更现代的数据写法：数据集中管理，多个 series 共享同一份数据，用 encode 指定哪列映射到哪个轴。

dataset: {
  source: [
    ['产品', '销售额', '利润'],   // 第一行是列名
    ['手机',    820,    120],
    ['电脑',    932,    200],
    ['平板',    601,     80],
  ]
},
series: [
  { type: 'bar',  encode: { x:'产品', y:'销售额' } },
  { type: 'line', encode: { x:'产品', y:'利润'  } },
]
// 两个 series 共享同一份 dataset，数据只写一次
💡推荐用 dataset：数据与图表逻辑分离，改数据不用改 series，多 series 共享同一份数据来源，维护更方便。适合从 CSV/API 加载数据的场景。
05

事件交互 —— chart.on()

ECharts 提供丰富的图表级事件（不是 DOM 事件），点击柱子直接拿到该柱子的完整数据，不需要自己解析坐标。

// 点击图表元素
chart.on('click', (params) => {
  console.log(params.name)    // 被点击项的名称
  console.log(params.value)   // 被点击项的数值
  console.log(params.seriesName) // 系列名称
})

// 悬停高亮
chart.on('mouseover', (params) => { ... })

// 动态更新数据（直接 setOption 增量合并）
chart.setOption({ series: [{ data: newData }] })
// ECharts 自动做动画过渡，不需要手写 transition
👆 点击任意扇形查看数据
💡ECharts 事件 vs D3 事件：D3 用原生 DOM 事件，回调里拿 d（绑定数据）；ECharts 用图表级事件，回调里拿 params 对象，直接包含 name/value/dataIndex 等，更方便。
06

样式与主题定制 —— itemStyle / theme

ECharts 提供三个层次的样式控制：全局主题 → series 级别 → 单项级别，越靠后优先级越高。

// 1. 初始化时传入内置主题（'dark' 是内置主题）
const chart = echarts.init(dom, 'dark')

// 2. series 级别：整体样式
series: [{
  type: 'bar',
  itemStyle: {
    color: '#3b82f6',           // 统一颜色
    borderRadius: [4,4,0,0],  // 顶部圆角
  },
  label: { show: true, position: 'top' }  // 显示数据标签
}]

// 3. 单项级别：给某一根柱子特殊颜色
data: [
  120, 200,
  { value:350, itemStyle:{ color:'#f97316' } }, // 高亮这一项
  80, 170
]
💡ECharts 还支持渐变色：color: new echarts.graphic.LinearGradient(0,0,0,1,[{offset:0,color:'#3b82f6'},{offset:1,color:'#1d4ed8'}])，可以做出从上到下的渐变柱子。

掌握 ECharts 的学习路径与技术栈

ECharts 入门门槛低，但要用好仍需系统学习。以下是从零到能独立开发业务可视化项目的完整路径。

阶段一 基础前置（门槛极低） 建议时长：1–2 周
技术需要掌握到什么程度
HTML会写一个有宽高的 <div> 就够了，ECharts 在里面渲染 Canvas/SVG
CSS能控制 div 的宽高、布局，不需要了解 SVG
JavaScript（基础）会声明变量、写对象字面量 { }、调用函数。不需要掌握数组高阶方法或异步——比 D3 低得多
✅ 完成阶段一 = 能把官方示例复制下来跑通
阶段二 ECharts 核心配置（能自己写图表） 建议时长：2–4 周
模块需要掌握到什么程度
option 结构理解 title / legend / tooltip / xAxis / yAxis / series 各自的职责，能不看文档手写基础配置
常用图表类型熟练写出：bar / line / pie / scatter；了解：radar / heatmap / funnel / gauge
坐标轴会区分 category / value / time 轴，会格式化标签，会处理双 Y 轴
tooltip / legend会自定义 tooltip 格式（formatter 函数），会控制图例显示/隐藏
dataset + encode会用 dataset 管理数据，会用 encode 做列映射，这是规范写法
事件会用 chart.on('click', params => {}) 做点击响应，会 setOption 动态更新数据
✅ 完成阶段二 = 能独立完成业务报表需求，不需要每次翻文档
阶段三 进阶技术栈（做复杂/大屏/工程化项目） 按需学习
技术/模块需要掌握到什么程度
地图（geo/map）会加载 GeoJSON，会做省市下钻，会配 visualMap 颜色映射
大数据渲染了解 sampling: 'lttb'（折线抽样）、large: true（散点大数据模式），知道何时用 Canvas 渲染器
自定义系列能用 series.type:'custom' + renderItem 画 ECharts 没有的图形
数据大屏了解 window.addEventListener('resize', chart.resize) 响应式，会用 CSS 缩放方案（scale transform）做大屏适配
Vue/React 集成会用 echarts-for-react 或 vue-echarts，理解组件生命周期与图表 init/dispose 的配合
自定义主题会用 Theme Builder 工具生成主题 JSON，会注册并使用自定义主题

⚡ 核心配置项速查表

配置项作用常用值/备注
title图表标题text / subtext / left / top
tooltip悬停提示框trigger: 'axis'（轴触发）/ 'item'（项触发）
legend图例自动读取 series.name，data 可手动指定
xAxis / yAxis坐标轴type: 'category'/'value'/'time'/'log'
series数据系列（核心）type 决定图表类型，data 放数据
dataset集中管理数据source 放二维数组，encode 映射列到轴
visualMap数据 → 颜色/大小映射热力图/地图必用，continuous/piecewise 两种
dataZoom数据缩放/平移type:'slider'（滑块）/'inside'（鼠标滚轮）
grid绘图区域边距left/right/top/bottom，防标签被裁剪
color全局调色板颜色数组，series 依次取色
